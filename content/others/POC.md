## POC 功能列表（国产芯片 + AliOS）

### 1. 应用层

| 模块            | POC 功能点          | 验证目的                |
| ------------- | ---------------- | ------------------- |
| Launcher / 桌面 | 应用入口、状态展示        | 验证基础桌面形态与系统调用稳定性    |
| 车控 / 车设       | 空调控制、基础车辆状态显示    | 验证车控能力在国产 OS 下的可运行性 |
| 多媒体           | 本地音频播放、简单视频播放    | 验证多媒体链路与系统资源占用      |
| 设置            | 网络 / 蓝牙 / 系统设置入口 | 验证系统配置与服务调用能力       |


### 2. 应用框架层

| 模块     | POC 功能点                                  | 验证目的                         |
| ------ | ---------------------------------------- | ---------------------------- |
| 应用开发接口 | CAF / NDK / MiniApp API                  | 验证非 Android Framework 的开发可行性 |
| UI 框架  | Page / View                              | 验证基础 UI 框架能力                 |
| 系统能力   | Notification / Provider                  | 验证系统级交互与数据访问                 |
| 系统服务接口 | Network / Account / Vehicle / Multimedia | 验证应用与系统服务解耦能力                |
| 输入与感知  | Input / Speech Service（基础）               | 验证输入与语音基础能力                  |


### 3. 系统服务层

| 模块    | POC 功能点                    | 验证目的         |
| ----- | -------------------------- | ------------ |
| 通信    | IPC                        | 验证系统内部通信机制   |
| 连接能力  | BT / BLE / WiFi            | 验证基础连接与设备管理  |
| 多媒体设备 | Audio / Camera（如支持）        | 验证设备驱动与服务稳定性 |
| 图形    | Graphic                    | 验证显示与渲染基础能力  |
| 系统能力  | Persistent / Network / adb | 验证系统可运维与调试能力 |


### 4. 车载服务层

| 模块               | POC 功能点           | 验证目的             |
| ---------------- | ----------------- | ---------------- |
| Car Service      | 车控信号通路            | 验证 OS 与车辆接口的集成能力 |
| Power Service    | 电源状态感知            | 验证电源管理与状态切换      |
| OTA Service      | 版本信息 / 升级流程验证（简化） | 验证升级链路（非量产）      |
| Diagnose Service | 基础状态上报            | 验证系统诊断与监控能力      |


### 5. AI 服务层

| 模块             | POC 功能点         | 验证目的                 |
| -------------- | --------------- | -------------------- |
| LLM 推理服务       | 本地小模型或云端调用      | 验证 AI 在国产 OS 下的运行可行性 |
| AI Service API | 统一调用接口          | 验证 AI 作为系统级服务的集成方式   |
| 能力调用           | AI 调用车控/多媒体示例   | 验证“AI 调用能力”闭环        |
| 资源评估           | CPU / 内存 / 延迟统计 | 评估 AI 对系统资源与稳定性的影响   |


### 6. 运行时与兼容层

| 模块             | POC 功能点                        | 验证目的                   |
| -------------- | ------------------------------ | ---------------------- |
| Runtime        | Page Manager / Mini App Engine | 验证原生应用运行能力             |
| Web            | Web Engine                     | 验证 Web 应用承载能力          |
| Android 兼容（可选） | 容器 / VM 运行简单 Android Demo      | 验证 Android 作为生态兼容层的可行性 |


### 7. 安全与系统可信

| 模块    | POC 功能点                                 | 验证目的        |
| ----- | --------------------------------------- | ----------- |
| 权限与身份 | Signature / Permission / Identification | 验证基础安全模型    |
| 内核安全  | AliSELinux / Kernel Hardening           | 验证系统可信与隔离能力 |
| 安全容器  | Security Container                      | 验证服务隔离与边界   |
| 网络安全  | Network Access Control                  | 验证网络访问控制能力  |


### 8. HAL 与底层（HAL / Kernel）

| 模块               | POC 功能点                 | 验证目的               |
| ---------------- | ----------------------- | ------------------ |
| HAL              | Audio / Display / Input | 验证 OS 与硬件抽象层适配     |
| Vehicle IF       | MCU / CAN 信号桥接          | 验证车身信号通路           |
| Kernel & Drivers | 启动、驱动加载、稳定性             | 验证国产芯片 + OS 的基础可用性 |
