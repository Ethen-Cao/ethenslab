①注册环回 Mix（控制面）

守护进程启动后调用 AudioSystem::registerPolicyMixes() 注册一个 LOOP_BACK | RENDER 的 AudioMix（匹配 usage = MEDIA/GAME/UNKNOWN）。因 is_mix_loopback_render()==true，跳过 MODIFY_AUDIO_ROUTING 权限校验。

②应用播放音频

各 App 通过 AudioTrack 把 PCM 写入 AudioFlinger 的 PlaybackThread/Mixer（蓝色流）。这是普通的播放路径，与本服务无关。

③分叉 = 复制一份

AudioPolicy 已把 Mixer 配置为：匹配的播放流同时走两条路 —— RENDER 照常进主 HAL 出扬声器，LOOP_BACK 复制到 REMOTE_SUBMIX 输出线程。播放不中断。

④环回 → 捕获

副本经 Remote Submix HAL 环回到 REMOTE_SUBMIX 输入线程；audio_capture_service 的 AudioRecord.read() 取走 PCM（橙色流），喂给 rhythm_analyzer 做节奏可视化。

依据：audio_capture_service.cpp（buildMix / createRecord / captureLoop）、README.md（绕过 MediaProjection 的 5 处源码证据）、DEBUG.md（Output→REMOTE_SUBMIX 复制 track 的 dumpsys 验证）。 注：未声明 allowAudioPlaybackCapture="true" 的 App，在 kCapturePrivileged=false 下不会被复制到 REMOTE_SUBMIX。
